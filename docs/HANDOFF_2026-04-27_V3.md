1. **Never introduce Tailwind grayscale tokens** (`text-zinc-*`, `bg-gray-*`, `border-neutral-*`, etc.). Currently zero violations in `src/`. Use `--v3-ink-*`, `--v3-bg-*`, `--v3-line-*`, or the V3-aliased `--color-text-*` family.
2. **Never `npm install` anything** — the existing tree is pinned and the operator's auto-commit could strip it.
3. **Never commit unless explicitly asked.** Operator has aggressive auto-commit; let it sweep.
4. **Don't add `--turbopack` back to dev script** until the OneDrive issue is resolved (see Open issues B).
5. **Don't touch `apps/trendingrepo-worker/`** — broken upstream, parked.
6. **Don't `rm -rf .next`** during a session — Turbopack/webpack rebuild from cold-start is slow on OneDrive and may trigger `_buildManifest.tmp` race conditions.
7. **Don't bypass `parseBody` / `serverError` / `errorEnvelope`** in API routes — those are the canonical helpers per APP-02/APP-03/APP-10 audit items.
8. **Don't import `@/lib/data-store` directly in any client component** — use the dynamic-import pattern in `bluesky.ts` etc., OR pass the data through props from a server component.
